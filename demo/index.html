Backbone: A Responsive Framework

Details

Backbone is a simple to use HTML, LESS/CSS and JavaScript/jQuery responsive framework (or foundation/boilerplate) to help with rapid protyping and development of websites for use on Desktop, Laptops, Tablets and Mobile devices. As standard it includes a few default files and plugins, such as 'HTML5 Shiv', 'normalize.css', 'FontAwesome', 'respond.js', 'smoothscroll.js', 'placeholders.js', 'Responsive Tabs', 'Responsive Accordion' and 'Animations'. Of course any of these files and plugins can be removed by simplying unlinking them to remove any unwanted bloat/load time. It also includes the Facebook API, Facebook Opengraph and Twitter Card HTML snippets, links to fav icons and apple icons in the 'assets/imgs' directory. As well as the conditional statement classes on the 'html' element.

Since most websites nowadays are accessed via portable devices such as Tablets and Mobiles, the whole Framework is built using CSS Media Queries. Since media queries do not work in older browsers such as Internet Explorer 8 and below we are using the 'respond.js,' polyfill as a fallback.

Usage

Simply copy the 'backbone' directory to your workflow area, rename the 'backbone' directory to the project you are working on and start developing from within this directory using the 'index.html' page as your base. To use in your root directory, copy the files from within the 'backbone' directory into your root directory.

Developing

The Framework uses LESS for its ease of use and in-browser compiling in the development process, as recommended by the creators of LESS themselves, it is advised to pre-compile your LESS to CSS when your site is in production mode. The 'style.less' file in the assets folder has various variables you can adjust to completely change the color scheme of your design such as document/browser color values, typography and forms etc.

All commonly used block elements default bottom margin is set to 20 pixels, these can adjusted by simply using the classes: bm-remove(0px), bm-smaller (5px), bm-small (10px), bm-medium (Default: 20px), bm-large (30px) and bm-larger (40px). If you wish to adjust the top margin of these elements as well, you can use the same class structure with one slight difference which is by altering 'bm' (Bottom Margin) to 'tm' (Top Margin).

It also uses the Box Model Fix which may require that you adjust to this style of programming and structuring if you haven't used it before.

There are various types of what we are calling 'quickers', which are handy for easily adjusting and styling elements, things such as text alignment, text transformations/variants, element floating, element display, indentation and cursor symbols. Font sizes can also be adjusted slightly by using the 'span' element and applying any of the font adjustment classes, this also works on header elements. All of these classes can be found in the 'style.less' file under the '#Quickers' section.

Fixed Grid

Fixed grid layouts are used through the one-half, one-quarter, two-quarter, three-quarter, one-third and two-third classes. A container element must wrapped around these classes using the container class. For the last column, the .last class should be applied to stop any page/template breakages. Grid elements stack vertically when the viewport is less than 768 pixels.

Percentage Grid

Percent grid layouts are used through the one-half-percent, one-quarter-percent, two-quarter-percent, three-quarter-percent, one-third-percent and two-third-percent classes. For the last column, the .last class should be applied to stop any page/template breakages. Percentage grids differ from fixed grids in a few ways, first of all they are fluid to the containing element, meaning they can be placed within any block level element. They use padding instead of margin to give a 20 pixel space which is why they seem to merge into one element, this has only happened since we have colorized the percentage grid in this demo. Another difference is they do not stack vertically the same way as fixed grids do, they will continue to take up their respective percentage of their parent element unless stated otherwise. To stack vertically the tablet or mobile class needs to be applied, depending on what device you wish for the elements to start stacking vertically.

Forms

Input, select and textarea elements come at a default width of 300 pixels, to adjust this you can apply either the w-small, w-large or w-full class which adjusts the elements width to 180 pixels, 420 pixels or 100% (of it's container) respectively. You can also increase the height by applying either the h-large or h-larger class which adjusts the height from its default value of 30 pixels to 40 pixels or 50 pixels respectively. Textarea heights for these classes are different from input and select elements, textareas by default are 80 pixels and are increased to 120 pixels or 160 pixels respectively. All these elements also stack vertically when the viewport is less than 768 pixels.

Buttons (as well as elements using the button class) can have the same classes applied as input, select and textarea elements, except butons have a default width of 125 pixels, and the classes adjusts the width to 100 pixels, 150 pixels or 100% (of it's container) respectively. The same classes for the buttons height are also the same as input, select and textarea elements and take the same height values as input and select elements. If, for example you have the w-small class applied to a button but the placeholder text spans larger than the width of the button, the button will increase to display the placeholder text inline. Button elements also stack vertically but not until the viewport is less than 480 pixels.
